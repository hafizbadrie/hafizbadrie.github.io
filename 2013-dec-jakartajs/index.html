Why React

"React is a JavaScript library for creating user interfaces by Facebook and Instagram. Many people choose to think of React as the V in MVC."

"Building large applications with data that changes over time"

  • Break application into components, not templates
  • Re-render whole app on every update
  • Virtual DOM

Component - Code

        
/** @jsx React.DOM */
var ChatBox = React.createClass({
  render: function() {
    var chat_line = function(sent_chat) {
      return <li>{sent_chat.itemText} <div id={sent_chat.itemId}>{sent_chat.commandResult}</div></li>;
    }
    return <ul id="chats">{this.props.items.map(chat_line)}</ul>;
  }
});

Virtual DOM

"When the data changes, React conceptually hits the "refresh" button, and knows to only update the changed parts."

But why? Isn't it expensive?

  • ReactJS component is loosely coupled
  • It's the easiest way to handle data changes
  • Yes, it's expensive, but that's when Virtual DOM taking action
